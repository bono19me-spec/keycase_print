# 키커버 인쇄 PDF 생성기 제작 가이드

## 1. 프로젝트 목적

단체예약 시스템에서 추출한 엑셀 파일을 사용자가 편집하지 않고 그대로 업로드하면, 기성 키커버 용지에 인쇄할 이름과 방번호만 포함된 PDF를 자동 생성하는 웹앱을 만든다.

이 앱은 Word의 메일병합을 대체하는 단순 업무 도구이다.

사용자는 다음 흐름으로 사용한다.

1. 단체예약 엑셀 파일 업로드
2. 데이터 자동 추출
3. 미리보기 확인
4. 테스트 PDF 생성
5. 필요하면 위치 조정
6. 지정 장수 단위로 PDF 생성
7. 키커버 용지를 수동 급지 트레이에 넣고 인쇄

---

## 2. 사용 환경 가정

- 사용자는 호텔 프런트 직원이다.
- 회사 PC에서 웹브라우저로 사용한다.
- 엑셀 파일은 단체예약 시스템에서 추출한 원본 파일을 그대로 사용한다.
- 키커버 용지는 기성품이다.
- 키커버 용지는 복사기/복합기의 수동 급지 트레이, 즉 手差しトレイ에 넣어서 인쇄한다.
- 커스텀 용지 트레이에는 많은 장수를 한 번에 넣을 수 없으므로, 20장 단위 등으로 나누어 출력할 수 있어야 한다.

---

## 3. 입력 파일

### 지원 파일 형식

- `.xls`
- `.xlsx`

### 샘플 파일 구조

업로드된 샘플 파일 기준:

- 시트명: `団体メンバ一覧表`
- 데이터 시작 행: 13행
- 방번호: U열
- 숙박자명: X열

### 추출 규칙

기본 규칙:

```text
13행부터 아래로 읽는다.
U열에서 방번호를 읽는다.
X열에서 숙박자명을 읽는다.
방번호와 숙박자명이 모두 비어 있는 행은 제외한다.
```

### 이름 처리 규칙

일본 호텔 업무 정서상 `様`를 붙여 출력한다.

```text
숙박자명 문자열 앞뒤 공백 제거
끝이 "様"이면 그대로 사용
끝이 "様"이 아니면 " 様" 또는 "様"을 붙임
```

기본값은 원본처럼 자연스럽게 보이도록 `공백 + 様`를 추천한다.

예:

```text
KURIHARA DAISHIKE 様 → 그대로 사용
KURIHARA DAISHIKE → KURIHARA DAISHIKE 様
山田太郎 → 山田太郎 様
```

## 4. 키커버 용지 정보

### 실제 용지 크기

```text
가로: 130mm
세로: 110mm
```

### 접는 방식

- 가로 방향으로 반으로 접어서 사용한다.
- 오른쪽 절반이 실제 앞면 인쇄 영역이다.
- 왼쪽 절반에는 인쇄하지 않는다.

### 인쇄 영역

```text
전체 PDF 페이지: 130mm × 110mm
오른쪽 인쇄 영역: X 65mm ~ 130mm, Y 0mm ~ 110mm
오른쪽 영역 중심 X: 97.5mm
```

### 주의

기성 키커버에는 이미 다음 요소가 인쇄되어 있다.

- 파란 대각선
- 東横INN 로고
- Room No. 문구

따라서 PDF에는 배경, 로고, 선, 접는 선, 꽃무늬, 테두리를 넣지 않는다.

PDF에는 이름과 방번호 텍스트만 출력한다.

---

## 5. PDF 출력 사양

### 기본 PDF 사양

```text
페이지 크기: 130mm × 110mm
1명당 1페이지
배경 없음
이미지 없음
테두리 없음
접는 선 없음
텍스트만 출력
```

예:

```text
1페이지: 704 / KURIHARA DAISHIKE 様
2페이지: 705 / NAKAMOTO AKIHITO 様
3페이지: 706 / DAIKOKU YOSHINORI 様
```

## 6. 텍스트 배치 기본값

사진을 참고한 첫 테스트용 기본 좌표이다. 실제 출력 후 조정 가능해야 한다.

좌표 기준:

```text
PDF 왼쪽 위를 원점으로 한다.
단위는 mm이다.
X는 왼쪽에서 오른쪽으로 증가한다.
Y는 위에서 아래로 증가한다.
```

### 이름 위치

```text
nameX: 97.5mm
nameY: 38mm
nameAlign: center
nameFontSize: 12pt 또는 13pt
```

### 방번호 위치

```text
roomX: 86mm
roomY: 88mm
roomAlign: left 또는 center
roomFontSize: 16pt 또는 18pt
```

방번호는 기존 기성 커버의 `Room No.` 문구 아래에 들어가야 한다.

## 7. 위치 조정 기능

실제 복사기 인쇄에서는 1~3mm 정도 위치가 밀릴 수 있다. 사용자가 코드 수정 없이 화면에서 조정할 수 있어야 한다.

### 필수 조정 항목

```text
전체 X 보정값: globalOffsetX, 단위 mm
전체 Y 보정값: globalOffsetY, 단위 mm
이름 X 위치: nameX, 단위 mm
이름 Y 위치: nameY, 단위 mm
방번호 X 위치: roomX, 단위 mm
방번호 Y 위치: roomY, 단위 mm
이름 글자 크기: nameFontSize
방번호 글자 크기: roomFontSize
180도 회전 여부: rotate180
```

### 사용 예

```text
전체가 오른쪽으로 밀림 → globalOffsetX = -2
전체가 왼쪽으로 밀림 → globalOffsetX = +2
전체가 아래로 밀림 → globalOffsetY = -2
전체가 위로 밀림 → globalOffsetY = +2
```

## 8. 대량 출력 안정성 기능

커스텀 용지/手差しトレイ에는 많은 키커버 용지를 한 번에 올릴 수 없다. 따라서 전체 PDF 하나만 만드는 방식은 피하고, 장수 또는 범위를 선택해서 PDF를 생성할 수 있어야 한다.

### 필수 출력 옵션

1. 테스트 1장 PDF 생성
2. 전체 PDF 생성
3. 범위 지정 PDF 생성
4. 지정 장수 단위 분할 PDF 생성

### UI 예시

```text
[테스트 1장 PDF 생성]
[전체 PDF 생성]
[범위 지정 PDF 생성]
[20장씩 분할 PDF 생성]
```

### 범위 지정 예시

```text
시작 번호: 1
끝 번호: 20
→ 1번째 숙박자부터 20번째 숙박자까지 PDF 생성
```

```text
시작 번호: 21
끝 번호: 40
→ 21번째 숙박자부터 40번째 숙박자까지 PDF 생성
```

### 분할 출력 예시

총 54명일 경우, 20장 단위로 분할하면 다음 파일을 만든다.

```text
keycover_001_020.pdf
keycover_021_040.pdf
keycover_041_054.pdf
```

### 분할 단위 선택

사용자가 직접 선택할 수 있게 한다.

```text
분할 단위:
- 5장
- 10장
- 20장
- 30장
- 직접 입력
```

기본값은 20장으로 한다.

## 9. 미리보기 기능

### 데이터 미리보기

엑셀 업로드 후 추출된 데이터를 표로 보여준다.

표시 항목:

```text
번호
방번호
숙박자명
출력 이름
```

예:

```text
1 | 704 | KURIHARA DAISHIKE 様 | KURIHARA DAISHIKE 様
2 | 705 | NAKAMOTO AKIHITO 様 | NAKAMOTO AKIHITO 様
```

### PDF 위치 미리보기

화면상에서 130mm × 110mm 비율의 미리보기 영역을 보여준다.

단, 실제 PDF에는 가이드선이나 테두리를 출력하지 않는다.

미리보기 화면에서는 사용자의 이해를 돕기 위해 선택적으로 다음을 표시해도 된다.

```text
용지 외곽선
오른쪽 인쇄 영역
이름 위치
방번호 위치
```

실제 PDF에는 이 선들을 절대 포함하지 않는다.

---

## 10. 테스트 출력 기능

실제 키커버를 낭비하지 않기 위해 테스트 기능이 필요하다.

### 테스트 PDF

- 첫 번째 숙박자 1명만 PDF로 생성한다.
- 또는 사용자가 선택한 1명을 PDF로 생성한다.
- 실제 PDF와 동일하게 텍스트만 출력한다.
- 기본적으로 가이드선은 출력하지 않는다.
- 필요하면 개발자용 옵션으로 가이드선 포함 PDF를 별도 제공해도 된다.

## 11. 인쇄 안내 문구

앱 화면에 아래 안내를 일본어로 표시한다.

```text
印刷時は必ず「実際のサイズ」または「100%」で印刷してください。
「ページに合わせる」や「用紙に合わせて縮小」は選択しないでください。
用紙サイズは 130mm × 110mm に設定してください。
手差しトレイを使用してください。
最初は1枚だけテスト印刷してください。
問題なければ10枚または20枚単位で印刷してください。
```

한국어 설명:

```text
인쇄 시 반드시 실제 크기 또는 100%로 인쇄한다.
페이지에 맞춤, 용지에 맞게 축소 옵션은 사용하지 않는다.
용지 크기는 130mm × 110mm로 설정한다.
수동 급지 트레이를 사용한다.
처음에는 1장만 테스트 출력한다.
문제가 없으면 10장 또는 20장 단위로 출력한다.
```

## 12. 권장 기술 구성

처음 버전은 서버 없이 브라우저에서 동작하는 정적 웹앱으로 만든다.

### 권장 구성

```text
HTML
CSS
JavaScript 또는 TypeScript
Excel 파싱 라이브러리
PDF 생성 라이브러리
```

### 권장 라이브러리 예시

- Excel 읽기: SheetJS `xlsx`
- PDF 생성: `pdf-lib` 또는 `jsPDF`

### 중요

가능하면 엑셀 파일을 서버로 업로드하지 않고 브라우저 안에서만 처리한다.

회사 업무 데이터가 포함될 수 있으므로, 다음 문구를 앱에 표시하면 좋다.

```text
このファイルはサーバーに送信されません。
ブラウザ内で処理されます。
```

## 13. 화면 구성 제안

### 화면 1: 파일 업로드

- 엑셀 파일 선택
- 지원 형식 안내
- 샘플 규칙 안내

### 화면 2: 데이터 확인

- 추출된 데이터 표
- 총 인원 수 표시

### 화면 3: 위치 설정

- 이름 위치 X/Y
- 방번호 위치 X/Y
- 전체 보정 X/Y
- 글자 크기
- 180도 회전
- 설정 초기화 버튼

### 화면 4: PDF 생성

- 테스트 1장 PDF 생성
- 범위 지정 PDF 생성
- 전체 PDF 생성
- 20장씩 분할 PDF 생성
- 분할 단위 선택

## 14. 오류 처리

### 엑셀 파일 오류

다음 경우 사용자에게 알린다.

```text
시트 "団体メンバ一覧表"를 찾을 수 없음
U열 또는 X열 데이터가 없음
13행 이후 유효한 데이터가 없음
파일 형식을 읽을 수 없음
```

### 데이터 오류

다음 행은 제외하거나 경고 표시한다.

```text
방번호가 비어 있음
이름이 비어 있음
방번호와 이름이 모두 비어 있음
```

사용자는 오류 행을 제외하고 PDF를 생성할 수 있어야 한다.

## 18. 개발 시 주의사항

- PDF에는 절대 배경 이미지를 넣지 않는다.
- 기성 커버에 이미 인쇄된 Room No. 아래에 방번호가 오도록 한다.
- 이름은 상단 빈 공간에 출력한다.
- 접는 선은 실제 PDF에 출력하지 않는다.
- 출력 좌표는 반드시 mm 단위로 관리한다.
- 인쇄 시 100% 크기로 출력해야 한다는 안내를 앱에 표시한다.
- 대량 출력 시 복사기 급지 문제를 줄이기 위해 20장 단위 분할 출력을 지원한다.
